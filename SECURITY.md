# Security Policy: AXON

## Supported Versions
Only the latest version of AXON is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability
We take the security of our sovereign infrastructure seriously. If you discover a vulnerability, please do not open a public issue. 

Instead, please report security vulnerabilities via [INSERT-PRIVATE-REPORTING-METHOD, e.g., Tuta/Proton Mail]. 

### Our Commitment:
* We will acknowledge receipt of your report within 48 hours.
* We will provide an estimated timeframe for a fix.
* We will coordinate a public disclosure after the fix is deployed.

## Security Principles
AXON follows a **Zero-Trust** architecture. Any contribution affecting the core orchestrator or communication protocols will undergo a mandatory security audit.
